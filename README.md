Kimi: # 📋 计算机图形学从0到就业：18个月完全学习清单

**基于您的情况（211科班 + C++基础 + 工业经验 + 低强度工作）**，这份清单每天投入**1.5-2小时**，周末**4-8小时**，标记✅处为**必须完成的里程碑**。

## 📐 **阶段0：数学基础唤醒（0-6周）**

### **目标**：不追求证明，只求"理解几何意义 + 会算"

**学习资源**（按优先级）：
1. **《3D Math Primer for Graphics and Game Development》**（第2版）
   - **必读章节**：1-8章（坐标系、向量、矩阵、变换）
   - **学习方式**：每天1章，**手推所有公式**，用 Python/C++ 验证
   - **耗时**：4周

2. **计算机图形学数学速成课**（B站/YouTube）
   - 搜索关键词："GAMES101 线性代数回顾"
   - **目的**：用图形学视角复习，建立"数学→图像"的直觉
   - **耗时**：1周

3. **《Linear Algebra Done Right》**（选读，数学洁癖者）
   - **目的**：深入理解特征值/奇异值的几何意义（用于PCA降维、形变分析）
   - **耗时**：1周（只看第5、7章）

✅ **里程碑1（第6周）**：
- 不看书手写：MVP矩阵推导过程（Model-View-Projection）
- 实现：C++函数 `glm::mat4 CreateMVP(glm::vec3 pos, glm::vec3 rot, glm::vec3 scale, glm::vec3 cameraPos)`
- **验收**：能正确渲染一个旋转的立方体（为OpenGL阶段做准备）

## 🎮 **阶段1：OpenGL入门（7-16周）**

### **目标**：理解现代渲染管线，能用Shader实现经典效果

**学习资源**：
1. **LearnOpenGL.com**（官网，中文翻译完整）
   - **强制顺序**：初学者→入门→光照→模型加载→高级OpenGL
   - **代码规范**：每个教程**必须自己敲**，不允许复制粘贴
   - **耗时**：8周（每周1-2课）

2. **《OpenGL SuperBible》（第7版）**
   - **用途**：当手册查，特别是高级特性（Compute Shader、多线程渲染）
   - **耗时**：并行查阅，不专门花时间

3. **《OpenGL编程指南》（红宝书）**（选读）
   - **用途**：理解底层原理（状态机、上下文）
   - **耗时**：2周（快速浏览）

✅ **里程碑2（第12周）**：
- 项目：`LearnOpenGL` 的 "模型加载" 章节完整跑通
- 能用 AssImp 库加载一个工业零件的 3D 模型（STEP格式转.obj）
- **验收**：模型在窗口中旋转，有光照，能响应鼠标缩放

✅ **里程碑3（第16周）**：
- **个人项目**：实现"工业零件缺陷可视化Demo"
  - 加载零件模型
  - 在缺陷位置（x,y,z）渲染红色球体
  - 实现简单的UI（ImGui）显示缺陷列表
- **产出**：录屏 30 秒，发 GitHub，写 README

## 🔥 **阶段2：Vulkan实战（17-32周）**

### **目标**：理解现代API设计，掌握GPU驱动级优化

**学习资源**：
1.  **《Vulkan Tutorial》**  （vulkan-tutorial.com，有中文版）
   - **强制顺序**：每章必须实现，**尤其 Debugging 章节**
   - **关键**：理解 VkInstance、Device、Queue、Swapchain 的生命周期
   - **耗时**：8周（别求快，VkMemoryBarrier 之类概念要反复看）

2.  **《Vulkan Cookbook》**  （Pawel Lapinski）
   - **用途**：解决具体问题（多线程渲染、内存管理、同步）
   - **学习方式**：作为教程的补充，遇到问题再查

3. **NVIDIA Vulkan 示例代码**（GitHub: NVIDIA/nsight-vulkan）
   - **用途**：学习工业级代码规范
   - **关键**：看他们的 CMake 组织、Resource Management

✅ **里程碑4（第24周）**：
- **项目**：将阶段1的"缺陷可视化Demo"用 Vulkan 重写
- **要求**：
  - 支持 100 万个缺陷点（用 Instancing 渲染）
  - 帧率 > 60fps（使用 GPU Query 测量）
  - 内存占用 < 500MB（用 VMA 库管理内存）
- **验收**：录屏对比 OpenGL 版，展示性能提升

✅ **里程碑5（第32周）**：
- **个人项目**：`MiniRenderer` GitHub 仓库
  - 功能：Vulkan + ImGui + 模型加载 + 简单光照
  - 代码规范：Google C++ Style
  - 测试：CI 自动化构建（GitHub Actions）
- **产出**：README 写明架构、构建方式、录屏演示
- **目标**：50+ stars（去 Reddit r/vulkan 和 r/GraphicsProgramming 宣传）

## 🏗️ **阶段3：图形引擎架构（33-48周）**

### **目标**：理解引擎不是API调用堆砌，而是架构设计

**学习资源**：
1.  **《Game Engine Architecture》**  （Jason Gregory）
   - **必读章节**：第1-7章（引擎基础、资源管理、渲染引擎）
   - **重点**：理解 **Component-Entity-System (ECS)** 架构
   - **耗时**：6周

2.  **《Real-Time Rendering》（第4版）**  （虎书）
   - **用途**：当圣经供着，遇到光照/阴影/后处理问题时查
   - **耗时**：并行学习，不专门安排

3. **开源引擎源码阅读**（选择1-2个）：
   - **bgfx**：跨平台渲染库，代码量适中（推荐）
   - **Filament**：Google 移动端渲染引擎，现代C++
   - **DiligentEngine**：学习 Render Graph 设计
   - **耗时**：10周（每周读一个核心模块）

✅ **里程碑6（第40周）**：
- **设计文档**：手写 `MyIndustrialRenderer` 架构图
  - Resource Manager（纹理/模型/Shader）
  - Render Graph（Pass 依赖管理）
  - Scene Graph（场景树）
  - ECS（Entity-Component-System）
- **产出**：PDF 发 GitHub，展示设计能力

✅ **里程碑7（第48周）**：
- **引擎 Demo**：实现一个最小可运行版本
  - 能加载场景文件（JSON 格式）
  - 支持多光源（Directional + Point）
  - 阴影（PCF）
  - 后处理（Bloom）
- **产出**：录屏 1 分钟，展示"工业零件数字孪生"效果
